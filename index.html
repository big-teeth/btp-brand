Critical Color Rules

  • ✕ Never use #444, #555, #777, #888, #111, #222 — not in the palette. Map to nearest palette color.
  • ✕ Never use small red text on dark backgrounds. Red on dark only works at display/headline sizes (48px+).

Element Spec / Value Notes
Purpose Visual rhythm break Sits between sections as a pacing moment. No text, no labels, no captions. Photography only.
Build Method Section · CSS Grid · Cover images Add a section. Inside it add a container set to Advanced CSS Grid with 4 columns. Each cell gets a background image set to Cover. NOT a Wix gallery. NOT a Photoshop file.
Section Settings Apply Max Width: OFF · No padding Turn Apply Max Width OFF so images bleed full width. Set all section padding to 0. Height is a fixed px value set on the section directly.
Height ~180px desktop Tablet: ~140px. Mobile: ~120px. Adjust manually at each breakpoint. Enough presence without dominating.
Grid Asymmetric 4-cell · 3px gap One wide cell (2fr) + three narrow cells (1fr each). Alternate wide cell left/right between strips on the same page for variety.
Photo Tone Consistent within strip Either all color or all B&W within a single strip. Never mix. Different strips on the same page may use different treatments.
Photo Content Company life preferred Team moments, events, office, personality shots. BTS project photography acceptable as supporting content but shouldn't dominate a strip.
Strip Order Escalate in energy On pages with multiple strips, save the most unexpected or visually surprising photos for lower strips. Build anticipation as the page progresses.
Slides / Print Adaptable Translates to Google Slides pitch decks as a full-bleed photo row between content slides. Maintain asymmetric proportions and consistent photo tone per strip.

Guardrails Non-negotiable rules for every build

Never Do — Design

  • ✕ Rounded corners on anything (border-radius: 0)
  • ✕ Horizontal red rules under section titles
  • ✕ Left red accent bars on dark sections
  • ✕ Small red text on dark backgrounds
  • ✕ Red backgrounds except for CTA + statement bars
  • ✕ Colors outside the 7-color palette
  • ✕ Barlow Condensed for body copy
  • ✕ Barlow Regular for body copy (use Medium 500)

Never Do — Wix Studio

  • ✕ Scale Proportionally on text elements (use Wrap)
  • ✕ Skip Apply Max Width on any section
  • ✕ Set text opacity directly (use a darker hex instead)
  • ✕ Fixed heights on content containers
  • ✕ Percentage-based padding (use px only)
  • ✕ Galleries for precise layout (use CSS containers)
  • ✕ Hero section height: Scale Proportionally (use Fit to Screen)
  • ✕ Leave image alt text blank

Deviations from the PDF Brand Guide — Web versions are correct

Body Copy Weight

PDF says Regular (400). Web uses Medium (500). Always use Medium on web.

Red Rules & Accent Bars

PDF shows horizontal red rules under titles and left accent bars. Both removed from web permanently.

Special Elite Font

PDF references Nitti Typewriter. Web uses Special Elite. Same role, different font. Special Elite is correct for web.
